# وثيقة الامتثال الشاملة لثيمات سلة (100% Compliance Checklist) - الإصدار المطور

هذه القائمة مبنية على توثيق سلة الرسمي لضمان قبول الثيم من المرة الأولى.

---

## 1. هيكلية الصفحات الإجبارية (Required Pages)
يجب أن يحتوي الثيم على قوالب جاهزة ومصممة لكل من:
- [ ] **الصفحة الرئيسية (Home Page):** مع دعم المكونات الديناميكية.
- [ ] **صفحات المنتجات:** (قائمة المنتجات، تفاصيل المنتج الواحد).
- [ ] **صفحات العميل:** (الملف الشخصي، قائمة الطلبات، تفاصيل الطلب، المفضلة، التنبيهات).
- [ ] **صفحات المدونة:** (قائمة المقالات، تفاصيل المقالة).
- [ ] **صفحات الماركات:** (قائمة الماركات، منتجات الماركة).
- [ ] **الصفحات العامة:** (السلة، نظام الولاء، صفحة الشكر، الصفحات التعريفية، صفحة الهبوط).

## 2. معايير تجربة المستخدم (UX/UI Standards)
- [ ] **البحث الذكي:** يجب أن يكون الوصول للبحث سهلاً وسريعاً في الهيدر.
- [ ] **عناصر البطاقة:** بطاقة المنتج يجب أن تحتوي على (الاسم، السعر، الصورة، زر الإضافة، شارة التخفيض).
- [ ] **معرض الصور:** دعم التمرير (Slider) وتكبير الصور (Zoom) في صفحة المنتج.
- [ ] **الفلاتر والفرز:** يجب أن تعمل الفلاتر بشكل صحيح في صفحات التصنيفات والماركات.
- [ ] **وضوح حالة الطلب:** في صفحات العميل، يجب أن تكون حالة الطلب وتفاصيله واضحة جداً.

## 3. المعايير التقنية الصارمة (Strict Technical Rules)
- [ ] **Lighthouse Mobile Score:** 80/100 كحد أدنى.
- [ ] **Semantic HTML:** استخدام `header`, `footer`, `main`, `section`, `article` بشكل صحيح.
- [ ] **No Inline Styles:** الاعتماد كلياً على ملفات CSS أو Tailwind.
- [ ] **Error Handling:** إظهار رسائل خطأ واضحة (مثل: "المنتج غير متوفر").
- [ ] **Safe Code:** منع استخدام أي دوال تسبب ثغرات أمنية (مثل `eval`).

## 4. التوافق مع Twilight SDK
- [ ] استخدام `salla.cart.event` لتحديث السلة لحظياً.
- [ ] استخدام `salla.comment` لنظام التعليقات والتقييمات.
- [ ] استخدام المكونات الجاهزة (`<salla-add-product-button>`, `<salla-filters>`).
