# Electronics Vertical

هذا المستند يصف معيار المصنع لمتجر إلكترونيات داخل بيئة سلة المحلية.

الهدف ليس تقليد متجر بعينه، بل التأكد أن الثيم يستطيع التعامل مع قطاع إلكترونيات حقيقي: أجهزة عالية القيمة، مواصفات كثيرة، ضمان، نفاد مخزون، طلب مسبق، باقات، وإكسسوارات.

## ما الذي يجب أن يظهر؟

- تصنيفات واضحة: جوالات، لابتوبات، صوتيات، ألعاب، منزل ذكي، كاميرات، شاشات، وإكسسوارات.
- منتجات تحمل مواصفات تقنية مثل السعة، البطارية، الاتصال، الشاشة، أو التوافق.
- ضمان واضح ورسائل ثقة مناسبة لقيمة الأجهزة.
- حالات بيع متنوعة: خصم، نافد، طلب مسبق، bundle، ومنتج رقمي مثل ضمان ممتد.
- أقسام رئيسية تساعد المتسوق على المقارنة والترقية والبيع الإضافي.
- تجربة عرض تفاعلية آمنة مثل `product-flip` مع إبقاء السعر والرابط وزر السلة متاحين.

## fixture

الـ fixture الرسمي:

```text
electronics-rich
```

يفحصه المصنع عبر:

```bash
node factory.js fixtures gate electronics-rich
node factory.js vertical gate electronics
```

## specs

لتطلب هذا القطاع في ثيم:

```json
"verticals": {
  "electronics": {
    "required": true,
    "fixture": "electronics-rich"
  }
},
"experiences": {
  "product_flip": {
    "required": true,
    "slug": "device-spec-cards"
  }
}
```

## حدود السياسة

المسموح:

- عرض المواصفات الموجودة في بيانات المنتج أو إعدادات المكون.
- استخدام Custom Component محلي آمن لتقديم تفاصيل المنتج.
- إبراز الضمان والشحن الآمن كرسائل عرض داخل الثيم.

المشروط:

- البحث بالصورة، مقارنة أسعار خارجية، تمويل خارجي، أو أي API خارج سلة يجب أن يدخل عبر `integrations/` وقدرة مستقلة.

المرفوض:

- اختراع حالة ضمان أو خصم أو توفر لا تأتي من بيانات المنتج أو إعدادات التاجر.
- إخفاء السعر أو زر السلة داخل تفاعل بصري يجعل الشراء أقل وضوحا.
